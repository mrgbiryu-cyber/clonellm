# 구현 메모

> 문서 상태:
> 이 문서는 초기 구현 범위와 당시 메모를 정리한 문서다.
> 아래 `현재`, `다음 작업` 표현은 작성 시점 기준일 수 있으므로, 실제 우선순위는 `docs/decision-history.md`와 최신 작업 계획 문서를 우선한다.

## 1. 구현 범위 스냅샷

첫 구현은 아래 범위를 다룹니다.

- 공개 사이트맵 수집
- 페이지군별 seed URL 생성
- 대표 페이지 HTML 저장
- 링크/이미지/헤딩/스크립트/스타일시트 추출
- 크롤링 제한 경로 placeholder 처리

## 2. 스냅샷 기준 미포함

아래는 아직 구현되지 않았습니다.

- 실제 클릭 기반 팝업/모달 상태 수집
- 브라우저 렌더 이후 DOM 캡처
- desktop/mobile 동시 렌더 비교
- 컴포넌트 자동 분류
- Figma Page 반영

## 3. 당시 다음 작업 메모

다음 작업 우선순위는 아래와 같습니다.

1. Playwright 대체 또는 환경 보정 후 상태 수집 재개
2. popup/modal/overlay 분리 저장
3. page -> section -> component 정규화 규칙 고도화
4. 편집기 입력 구조와 실제 UI 연결

## 4. 스냅샷 기준 가능한 상태

작성 시점 기준으로는 아래까지 즉시 활용 가능했습니다.

1. 공개 사이트맵 기반 페이지군 추출
2. 대표 페이지 HTML/링크/이미지 아카이브
3. 이미지 자산 로컬 저장
4. page/section/component 후보 생성
5. 편집기 입력용 JSON 생성

즉, 당시 기준으로는 브라우저 기반 팝업 상태 수집을 제외한 정적 파이프라인이 실행 가능한 상태로 정리돼 있었다.
