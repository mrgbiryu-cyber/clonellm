# 관리자 화면 UX 상세 설계

> 상위 문서: `docs/admin-ux-structure-plan-2026-04-16.md`
>
> 이 문서는 세 섹션 각각의 **컴포넌트 레벨 표현 방식**을 다룬다.
> 구조 결정은 상위 문서 기준이며, 이 문서는 "어떻게 보여줄 것인가"만 다룬다.

---

## 1. 설계 기준

제미나이 제안과 이 프로젝트의 맥락을 비교해 아래 기준을 정한다.

| 제미나이 제안 | 이 프로젝트에서의 판단 |
|---|---|
| 섹션별 카드 레이아웃 | 채택 — 정체성 읽기 뷰, 기획서 섹션에 적용 |
| 좌측/상단 고정 네비 | 부분 채택 — 전체 3섹션용은 sticky anchor로 이미 설계됨. 기획서 내부 목차는 별도 추가 |
| 표와 리스트 강제 | 채택 — 기획서 출력, 요구사항 우선순위 항목에 적용 |
| Tailwind CSS 전환 | 미채택 — 현재 코드베이스는 커스텀 CSS 기반. 이번 범위에서 프레임워크 교체는 과도함. 대신 typography 규칙을 문서화하고 일관 적용 |
| 아이콘 + 시각 요소 | 채택 — 기획서 섹션 제목에 아이콘 배치. 이미지 placeholder는 이 도구 성격과 맞지 않아 제외 |

핵심 원칙:

- **정체성**은 "읽는 문서"처럼 보여야 한다. 폼처럼 보이면 안 된다.
- **요구사항**은 "채우는 폼"이다. 필드 간 그룹이 명확해야 한다.
- **기획서**는 "전달할 문서"다. 줄글이 아니라 구조화된 렌더링이어야 한다.

## 2. 페이지 정체성 — 표현 방식

### 2.1 읽기 뷰 (기본 상태)

정체성이 있는 경우 카드 그리드로 표시한다.

**레이아웃:**

```
┌─────────────────────────────────────────────┐
│ 페이지 정체성                    [편집]       │  ← 섹션 헤더
├──────────────────┬──────────────────────────┤
│ 페이지 역할       │ 브랜드 홈이자 전체 서비스  │  ← 2컬럼 카드
│ (라벨, 회색 소문) │ 진입 허브                 │
├──────────────────┼──────────────────────────┤
│ 디자인 의도       │ 신뢰감과 탐색 리듬을 함께  │
│                  │ 유지                      │
├──────────────────┴──────────────────────────┤
│ 페이지 목적                                   │  ← 전체 너비 카드
│ 이 페이지가 원래... (본문 텍스트)              │
├─────────────────────────────────────────────┤
│ 반드시 유지할 것         피해야 할 것          │  ← 2컬럼 분리
│ • 헤더 고정 내비         • 하단 광고 배너      │
│ • 브랜드 컬러 기준        • 팝업 과다          │
├─────────────────────────────────────────────┤
│ 시각 가드레일                                 │
│ • 여백 최소 24px • 폰트 계층 3단계 이하       │
└─────────────────────────────────────────────┘
```

**스타일 규칙:**

- 섹션 배경: `#f8fafc` (본문 흰색과 구분)
- 라벨: `font-size: 11px`, `color: #6b7280`, `font-weight: 700`, uppercase 또는 한글 소문자
- 값: `font-size: 14px`, `color: #111827`, `line-height: 1.6`
- 리스트형 항목 (유지할 것/피해야 할 것): bullet `•` 기호, 항목당 한 줄
- 비어 있는 필드: 값 자리에 `—` 표시, 회색 처리

**섹션 헤더:**

- 좌측: "페이지 정체성" + 상태 배지 (`ready` / `empty`)
- 우측: "편집" 버튼 (텍스트 버튼, 작게)

### 2.2 편집 뷰

"편집" 버튼 클릭 시 같은 자리에서 폼으로 전환한다. 모달 없음.

**필드 구성:**

| 필드 | 입력 유형 | 너비 | 비고 |
|------|-----------|------|------|
| 페이지 역할 | text input | 1/2 | placeholder: "예: 브랜드 홈이자 진입 허브" |
| 디자인 의도 | text input | 1/2 | placeholder: "예: 신뢰감과 탐색 리듬 유지" |
| 페이지 목적 | textarea | 전체 | min-height: 80px |
| 반드시 유지할 것 | textarea | 1/2 | "한 줄에 하나씩 입력" 안내 |
| 피해야 할 것 | textarea | 1/2 | "한 줄에 하나씩 입력" 안내 |
| 시각 가드레일 | textarea | 전체 | min-height: 64px |

**하단 액션:**

```
[취소]  [저장]
```

- "취소": 변경 사항 버리고 읽기 뷰 복귀
- "저장": 즉시 저장 후 읽기 뷰 복귀. 저장 중 버튼 비활성 + "저장 중…" 텍스트

### 2.3 빈 상태 뷰

```
┌─────────────────────────────────────────────┐
│ 페이지 정체성                                 │
├─────────────────────────────────────────────┤
│                                             │
│   아직 이 페이지의 정체성이 정의되지 않았습니다. │
│   정체성은 요구사항 작성과 기획서 생성의       │
│   기준이 됩니다.                              │
│                                             │
│             [정체성 작성하기]                 │
│                                             │
└─────────────────────────────────────────────┘
```

- 배경: `#fafafa`, 테두리: `border: 1px dashed #d1d5db`
- 버튼 클릭 시 바로 편집 뷰로 전환

## 3. 요구사항 — 표현 방식

### 3.1 폼 구조

요구사항은 항상 편집 가능한 폼이다. 필드를 4개 그룹으로 나눠 시각적으로 묶는다.

```
┌─────────────────────────────────────────────┐
│ 요구사항                                      │  ← 섹션 헤더
├─────────────────────────────────────────────┤
│ ▌ 기본 정보                                   │  ← 그룹 구분선 (왼쪽 4px bar)
│                                             │
│   요청 제목                                   │
│   [──────────────────────────────────────]  │
│                                             │
│   요청 배경                                   │
│   [──────────────────────────────────────]  │
│   [                                      ]  │
│                                             │
├─────────────────────────────────────────────┤
│ ▌ 방향 설정                                   │
│                                             │
│   기획 방향             디자인 방향            │
│   [──────────────]     [──────────────]     │
│   [              ]     [              ]     │
│                                             │
│   변화 강도                                   │
│   ○ 낮음  ● 보통  ○ 높음                     │
│                                             │
├─────────────────────────────────────────────┤
│ ▌ 제약 / 가드레일                              │
│                                             │
│   유지할 것             변경할 것              │
│   [──────────────]     [──────────────]     │
│   [              ]     [              ]     │
│                                             │
│   전체 가드레일                               │
│   [──────────────────────────────────────]  │
│                                             │
├─────────────────────────────────────────────┤
│ ▌ 우선순위 컴포넌트                             │
│                                             │
│   ┌──────────────┬──────────────────────┐  │
│   │ 컴포넌트      │ 이유                  │  │
│   ├──────────────┼──────────────────────┤  │
│   │ 히어로 배너   │ 첫 인상 결정 영역      │  │
│   │ 카테고리 탭   │ 탐색 흐름 시작점       │  │
│   └──────────────┴──────────────────────┘  │
│   [+ 항목 추가]                              │
│                                             │
├─────────────────────────────────────────────┤
│                    [저장]  [기획서 생성 →]    │
└─────────────────────────────────────────────┘
```

### 3.2 그룹 스타일 규칙

- 각 그룹 좌측에 4px 세로 바 (`border-left: 4px solid #e5e7eb`)
- 그룹 제목: `font-size: 12px`, `font-weight: 700`, `color: #6b7280`, 대문자 아님
- 그룹 배경: 모두 흰색. 그룹 사이 구분선 (`border-top: 1px solid #f3f4f6`)

### 3.3 우선순위 테이블

우선순위 항목은 줄글 textarea가 아니라 테이블 행으로 관리한다.

| 컴포넌트명 | 우선 이유 | 삭제 |
|-----------|-----------|------|
| text input | text input | × |

- 행 추가: "+ 항목 추가" 버튼
- 행 삭제: 각 행 우측 × 버튼
- 빈 상태: "아직 우선순위 컴포넌트가 없습니다" + "+ 항목 추가"

### 3.4 변화 강도 선택

radio 버튼 대신 세그먼트 선택 버튼으로 표시한다.

```
[  낮음  |  보통 (선택됨)  |  높음  ]
```

각 옵션 아래 보조 설명:
- 낮음: "구조 유지, 세부 조정"
- 보통: "주요 섹션 재편"
- 높음: "전체 흐름 재설계"

### 3.5 CTA 영역

폼 하단 고정 액션 바:

```
┌─────────────────────────────────────────────┐
│  마지막 저장: 2026-04-16 14:32    [저장]  [기획서 생성 →]  │
└─────────────────────────────────────────────┘
```

- "저장": 요구사항만 저장, 기획서 생성 없음
- "기획서 생성 →": 저장 후 즉시 기획서 생성 시작
- 기획서가 이미 있으면 "기획서 재생성 →"으로 라벨 변경
- 생성 중: 두 버튼 모두 비활성, "기획서 생성 →" 자리에 진행 표시

### 3.6 생성 중 상태

```
┌─────────────────────────────────────────────┐
│  기획서를 생성하고 있습니다...                │
│  ████████████░░░░░░░░░░░░░░░  45%           │
└─────────────────────────────────────────────┘
```

- 진행 바 + 퍼센트 또는 단계 텍스트
- 완료 시: "기획서 섹션으로 이동 →" 링크 포함 완료 메시지
- 실패 시: 에러 메시지 + "다시 시도" 버튼

## 4. 기획서 — 표현 방식

### 4.1 전체 레이아웃

기획서 섹션은 두 영역으로 나뉜다.

```
┌──────────────────────────────────────────────────────┐
│ 기획서          [직접 편집]  [내보내기 ▾]              │  ← 섹션 헤더
├──────────────────────────────────────────────────────┤
│                                                      │
│  ┌──────────────┐  ┌────────────────────────────┐   │
│  │ 목차          │  │ 본문 렌더링 영역             │   │
│  │ ─────────    │  │                            │   │
│  │ 1. 요약      │  │ (기획서 내용)               │   │
│  │ 2. 배경      │  │                            │   │
│  │ 3. 목표 범위  │  │                            │   │
│  │ 4. 기능      │  │                            │   │
│  │ 5. 사용자 흐름│  │                            │   │
│  │ 6. 로드맵    │  │                            │   │
│  │ 7. 미결 사항  │  │                            │   │
│  └──────────────┘  └────────────────────────────┘   │
└──────────────────────────────────────────────────────┘
```

- 목차: 좌측 고정, 너비 `160px`, 본문 스크롤 시 따라옴 (`position: sticky`)
- 본문: 나머지 너비 전체 사용

### 4.2 목차 스타일

```
  목차
  ──────
  ● 요약            ← active 상태 (현재 위치)
    배경
    목표 범위
    핵심 기능
    사용자 흐름
    로드맵
    미결 사항
```

- 항목: `font-size: 12px`, `color: #6b7280`
- active: `color: #111827`, `font-weight: 700`, 좌측 2px 바
- 클릭 시 해당 섹션으로 스크롤

### 4.3 본문 섹션 카드

기획서 각 섹션은 독립 카드로 렌더링한다.

```
┌─────────────────────────────────────────────┐
│ 📋  1. 요약                                   │  ← 아이콘 + 제목
├─────────────────────────────────────────────┤
│                                             │
│  LG전자 홈 페이지의 히어로 배너와 카테고리     │
│  탐색 흐름을 재편하여 첫 방문 전환율을        │
│  높이는 것을 목표로 합니다.                   │
│                                             │
└─────────────────────────────────────────────┘
```

**카드 스타일:**

| 섹션 | 배경색 | 아이콘 |
|------|--------|--------|
| 요약 | `#ffffff` | 📋 |
| 배경 / 문제 | `#fafafa` | 🔍 |
| 목표 범위 | `#ffffff` | 🎯 |
| 핵심 기능 | `#fafafa` | ⚡ |
| 사용자 흐름 | `#ffffff` | 🔄 |
| 로드맵 | `#fafafa` | 📅 |
| 미결 사항 | `#fffbeb` | ⚠️ |

- 카드 border: `1px solid #e9ebf5`
- 카드 border-radius: `10px`
- 카드 간격: `16px`
- 섹션 제목: `font-size: 16px`, `font-weight: 700`

### 4.4 핵심 기능 섹션 — 테이블 렌더링

핵심 기능은 테이블로 출력한다. 줄글 금지.

| 번호 | 기능명 | 설명 | 우선순위 |
|------|--------|------|---------|
| 1 | 히어로 배너 개편 | 첫 화면 메시지 구조 재편 | 높음 |
| 2 | 카테고리 탭 정렬 | 탐색 흐름 진입 단순화 | 보통 |

- 테이블 헤더 배경: `#f3f4f6`
- 우선순위 칩: 높음(빨강), 보통(파랑), 낮음(회색)

### 4.5 사용자 흐름 섹션 — 단계 리스트 렌더링

사용자 흐름은 화살표 단계 리스트로 출력한다.

```
① 홈 진입
    ↓
② 히어로 배너에서 주요 메시지 확인
    ↓
③ 카테고리 탭으로 탐색 시작
    ↓
④ 제품 리스트 → 상세 진입
```

### 4.6 로드맵 섹션 — 단계 카드

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  1단계        │  │  2단계        │  │  3단계        │
│  히어로 개편  │→ │  탭 정렬      │→ │  검증 / QA   │
│  완료 기준:   │  │  완료 기준:   │  │  완료 기준:   │
│  시안 확인    │  │  A/B 테스트   │  │  acceptance  │
└──────────────┘  └──────────────┘  └──────────────┘
```

- 3컬럼 그리드 (모바일: 1컬럼)
- 각 단계 카드에 완료 기준 한 줄

### 4.7 타이포그래피 규칙

기획서 본문 전체에 적용한다.

| 요소 | font-size | font-weight | color |
|------|-----------|-------------|-------|
| 섹션 제목 (h2) | 16px | 700 | #111827 |
| 소제목 (h3) | 14px | 700 | #374151 |
| 본문 | 14px | 400 | #374151 |
| 보조 설명 | 12px | 400 | #6b7280 |
| 테이블 헤더 | 12px | 700 | #374151 |
| 테이블 본문 | 13px | 400 | #111827 |

줄 간격: `line-height: 1.7` (본문 기준)

### 4.8 직접 편집 모드

"직접 편집" 버튼 클릭 시 렌더 뷰 → textarea 전환.

```
┌─────────────────────────────────────────────┐
│ 직접 편집 중          [미리보기]  [저장]  [취소] │
├─────────────────────────────────────────────┤
│ # 1. 요약                                    │
│ LG전자 홈 페이지의 히어로 배너와...            │
│                                             │
│ ## 2. 배경 / 문제 정의                        │
│ ...                                         │
└─────────────────────────────────────────────┘
```

- textarea: 마크다운 원문 편집
- "미리보기": 저장 없이 렌더 결과 확인
- 저장 후: "직접 수정됨" 배지 → 섹션 헤더 옆에 표시

### 4.9 내보내기

"내보내기 ▾" 클릭 시 드롭다운:

```
┌─────────────────┐
│ PDF 다운로드     │
│ 마크다운 복사    │
└─────────────────┘
```

- PDF: `window.print()` + 인쇄 전용 CSS 또는 html2canvas 방식
- 마크다운 복사: 원문 마크다운을 클립보드에 복사

### 4.10 빈 상태

```
┌─────────────────────────────────────────────┐
│ 기획서                                        │
```
